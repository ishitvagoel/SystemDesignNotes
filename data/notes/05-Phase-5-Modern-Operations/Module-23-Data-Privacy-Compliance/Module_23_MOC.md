# Module 23: Data Privacy & Compliance

*Designing for regulation, sovereignty, and data protection.*

## Overview
How to build systems that respect user privacy, satisfy global regulations (GDPR, HIPAA), and manage the complex lifecycle of sensitive data.

## Topics
- [[Data Privacy and Compliance]] — Regulatory frameworks, masking, and residency.
- [[Multi-Region E-Commerce]] — Scaling for global compliance.

## Learning Objectives
- Design for "Right to be Forgotten" with crypto-shredding.
- Understand data residency vs. data sovereignty.
- Implement PII vaults and tokenization layers.
- Architect audit trails for non-repudiation.
