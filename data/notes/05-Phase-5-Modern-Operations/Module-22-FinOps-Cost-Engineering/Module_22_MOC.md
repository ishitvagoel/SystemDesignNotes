# Module 22: FinOps & Cost Engineering

*Architecting for unit economics and cloud efficiency.*

## Overview
Cloud cost is a first-class engineering metric, just like latency or error rate. This module covers how to bridge the gap between finance and engineering to build cost-efficient distributed systems.

## Topics
- [[03-Phase-3-Architecture-Operations__Module-18-Multitenancy-Geo-Cost__Cost_Engineering_and_FinOps]] — The core principles of cloud cost management, FinOps culture, and the anatomy of a cloud bill.
- [[Cloud_Cost_Optimization]] — Purchasing models (savings plans, spot, reserved), right-sizing, cost allocation tagging, and the 30% waste rule.
- [[FinOps_Observability_and_Unit_Economics]] — Unit economics calculation, cost dashboards, anomaly detection patterns, showback vs chargeback.
- [[Resource_Right-Sizing_and_Autoscaling]] — Right-sizing methodology, HPA/ASG autoscaling strategies, capacity planning, and the autoscaling death spiral failure mode.
- [[Module_Module_18_MOC]] — Related multi-tenancy and geo-cost patterns.

## Learning Objectives
- Understand the "Frugal Architect" laws.
- Calculate cost crossovers for Serverless vs. Containers.
- Design for egress cost optimization in multi-region setups.
- Implement cost attribution and visibility across teams.
